# 解决方案 - ERR_CONNECTION_RESET 错误

## 问题分析
你遇到的错误：
```
POST http://localhost:8000/api/books net::ERR_CONNECTION_RESET
```

## 已验证的信息
✅ 后端服务正常运行（端口 8000 有多个进程监听）
✅ API 配置已更新为局域网 IP（192.168.8.199:8000）
✅ 后端可以正常响应请求（所有地址都测试通过）

## 根本原因
**小程序开发工具的网络请求限制**

微信开发者工具默认会阻止对非 HTTPS 和未备案域名的请求。即使在 `api.js` 中配置了正确的 IP 地址，也需要在开发工具中手动开启"不校验合法域名"选项。

## 解决步骤

### 方法 1: 配置开发工具（推荐）

1. **打开设置**
   - 在微信开发者工具中，点击右上角的 **详情** 按钮
   - 或者点击菜单栏的 **工具** > **设置**

2. **开启关键选项**
   在 **本地设置** 标签页中，勾选：
   - ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**
   - ✅ **开启调试模式**

3. **重启小程序**
   - 点击工具栏的 **编译** 按钮重新编译
   - 或者关闭小程序重新打开

4. **验证连接**
   - 访问测试页面：`/pages/test/test`
   - 或者尝试发布一本书

## 常见问题

### Q1: 为什么使用了 IP 还是不行？
A: 因为小程序开发工具默认阻止非 HTTPS 请求，必须开启"不校验合法域名"选项。

### Q2: 开启选项后仍然失败？
A: 请检查：
- 后端服务是否真的在运行
- 防火墙是否阻止了连接
- IP 地址是否正确
- 电脑和手机是否在同一网络（如果使用真机调试）

### Q3: 发布后怎么办？
A: 正式发布时需要：
1. 购买域名并备案
2. 配置 HTTPS 证书
3. 在小程序后台配置合法域名
4. 使用正式的域名替换 IP 地址
